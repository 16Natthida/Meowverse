# การติดตั้งและตั้งค่าฐานข้อมูล

โปรเจกต์นี้มี backend API สำหรับจัดการสินค้าและหมวดหมู่ โดยข้อมูลจะเก็บในฐานข้อมูล `meowverse`
ถ้าจะเอาโค้ดไปใช้เครื่องอื่น ให้เตรียม 3 อย่างหลัก ๆ ก่อน:

1. ติดตั้ง Node.js
2. ติดตั้ง MySQL
3. สร้างไฟล์ `.env` ของเครื่องนั้นเอง

## 1) สิ่งที่ต้องติดตั้งเพิ่ม

### 1.1 Node.js

ต้องใช้ Node.js เวอร์ชันที่โปรเจกต์รองรับ คือ `20.19+` หรือ `22.12+`

ถ้ายังไม่มี ให้ติดตั้ง Node.js ก่อน เพราะคำสั่ง `npm install`, `npm run dev`, และ `npm run api` ต้องใช้ Node กับ npm

### 1.2 MySQL

ต้องมีฐานข้อมูล MySQL ที่รันอยู่บนเครื่อง หรือเป็นเซิร์ฟเวอร์ที่เชื่อมต่อได้

โปรเจกต์นี้เชื่อมฐานข้อมูลผ่านค่าใน `.env` เช่น `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`

### 1.3 Git

ถ้าจะ clone โค้ดจาก GitHub เครื่องใหม่ ควรมี Git ติดตั้งไว้ด้วย

## 2) เตรียมฐานข้อมูล

1. สร้างฐานข้อมูลชื่อ `meowverse`
2. import ไฟล์ `meowverse.sql` ลงในฐานข้อมูลนั้น
3. ถ้าต้องการฟีเจอร์สินค้าผู้ดูแลระบบครบ ให้รัน migration เพิ่มได้ที่
   - `database/migrations/2026-04-16-admin-products.sql`

หมายเหตุ: backend มีระบบช่วยตรวจและสร้างคอลัมน์หรือ table ที่จำเป็นบางส่วนให้อัตโนมัติเมื่อเริ่มรัน API

## 3) ตั้งค่า environment

ให้คัดลอกไฟล์ `.env.example` เป็น `.env` แล้วแก้ค่าให้ตรงกับเครื่องของตัวเอง

หมายเหตุ: ในเอกสารนี้ไม่แสดงค่ารายละเอียดของ `.env` เพื่อความปลอดภัย ให้ดูโครงสร้างตัวแปรจากไฟล์ `.env.example` และตั้งค่าจริงเฉพาะในเครื่องของแต่ละคน

ข้อสำคัญ:

- ห้าม push ไฟล์ `.env` ขึ้น git
- ให้แชร์แค่ `.env.example` สำหรับเป็นตัวอย่างเท่านั้น

## 4) ติดตั้ง dependency ของโปรเจกต์

รันคำสั่งนี้ในโฟลเดอร์โปรเจกต์:

```bash
npm install
```

คำสั่งนี้จะติดตั้ง package ที่จำเป็นทั้งหมด เช่น Vue, Vite, Express, mysql2, multer, dotenv และตัวช่วย build/lint ที่โปรเจกต์ใช้อยู่

## 5) วิธีรันโปรเจกต์

### รันแยกทีละส่วน

เปิด backend API:

```bash
npm run api
```

เปิด frontend:

```bash
npm run dev
```

### รันพร้อมกันทั้ง frontend และ API

```bash
npm run dev:full
```

คำสั่งนี้เหมาะที่สุดเวลาพัฒนาในเครื่อง เพราะจะเปิดทั้งสองส่วนพร้อมกัน

## 6) ที่อยู่สำหรับทดสอบ

- Frontend: http://localhost:5173
- API health: http://localhost:3001/api/health
